# Phase 5 acceptance record

Generated July 17, 2026 Eastern Time.

## Outcome

PASS — `txline-kit-cpi` now provides a pinned, feature-locked Rust ABI and a return-safe three-argument Anchor CPI helper for TxLINE `validate_stat_v2`. It compiles for the Solana SBF target and from a fresh packaged consumer.

## ABI contract

- The source contract is pinned to TxODDS `tx-on-chain` commit `f7e3bcd5db4c6744445f75dfab7eccc879c6d2de`.
- The instruction discriminator is the exact IDL byte sequence `[208,215,194,214,241,71,246,178]`.
- Rust types preserve every V2 field order, signedness, enum order, vector, option, 32-byte hash, proof-node direction, score-stat leaf, and strategy variant.
- Three independent golden vectors were generated by the already working Anchor TypeScript client. Rust matches them byte for byte: basic home-win payload (170 bytes), all strategy variants (187 bytes), and populated directional proof paths (302 bytes).
- The daily score PDA uses the exact `daily_scores_roots` + u16 little-endian epoch-day seeds.

## CPI safety

- Mainnet is the default feature; devnet is an explicit no-default-features build.
- Enabling both networks or neither network is a compile-time error.
- `Program<'info, TxLine>` gives consumer account constraints a feature-selected program ID.
- The helper rejects a wrong program ID, non-executable program account, timestamp/summary mismatch, wrong daily-root PDA, missing return data, return data from another program, and malformed/trailing boolean bytes.
- Return data is read immediately after CPI and accepted only when the returning program equals the configured TxLINE deployment.

## Consumer surface

The intended call is:

```rust
let valid = validate_stat_v2_cpi(
    CpiContext::new(txline_program, ValidateStatV2 { daily_scores_merkle_roots }),
    payload,
    strategy,
)?;
```

The crate README includes a complete Anchor account/context example and devnet dependency configuration.

## Boundary

This phase proves ABI parity, package usability, Solana-target compilation, and defensive return decoding. It does not claim that an undeployed local test can execute TxLINE CPI on mainnet. The real consumer-program invocation, escrow settlement, and auditable mainnet transaction chain are Phase 6 gates. This library is integration tooling, not an audit.
